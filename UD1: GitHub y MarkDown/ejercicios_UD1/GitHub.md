## GitHub: Introducción
**Autora:** Beatriz Gutiérrez Escudero

### Introducción
GitHub es una plataforma basada en la web que permite el control de versiones y la colaboración en el desarrollo de software mediante Git. Su principal utilidad radica en facilitar el trabajo en equipo, el seguimiento de cambios y la gestión eficiente de proyectos de software.  

En este informe, se presentarán los pasos fundamentales para comenzar a utilizar GitHub, desde la creación de una cuenta hasta la gestión de ramas y colaboraciones. También se explicarán algunas de las funciones más importantes que ofrece la plataforma.

### Pasos para utilizar GitHub

#### Paso 1: Crear una cuenta y un repositorio en GitHub
 1. _**Crear una cuenta en GitHub:**_
   Dirígete a [https://github.com](https://github.com) y regístrate con tu correo electrónico.
<img width="1000" height="604" alt="crearCuentaGitHub" src="https://github.com/user-attachments/assets/d771d912-4280-4804-b6e4-2cea1a56879e" />

 2. _**Crear un nuevo repositorio:**_
    - Una vez iniciada la sesión, haz clic en el botón **"New" (Nuevo)** o en **"Create repository"**.
    <img width="1220" height="609" alt="nuevoRepositorio" src="https://github.com/user-attachments/assets/28a5d1cd-f16a-4018-b07c-2e5568982c41" />
   - Completa los siguientes campos:
     - **Repository name:** pon un nombre al repositorio (por ejemplo, `pruebaGitHub`).
     - **Description:** descripción breve del proyecto (opcional).
     - **Visibilidad:** selecciona público o privado (en este caso, público).
     - Marca la opción **"Initialize this repository with a README"** para comenzar con un archivo README.
   - Haz clic en **"Create repository"**.
<img width="775" height="571" alt="detallesRepositorio1" src="https://github.com/user-attachments/assets/f7dd6e9c-147f-4f90-9c2d-e98f8f5c0592" />
<img width="713" height="216" alt="detallesRepositorio2" src="https://github.com/user-attachments/assets/13b0cc12-ff1e-439d-ae90-1f3469053c40" />

#### Paso 2: Subir Archivos al Repositorio

1. **Ir al repositorio recién creado.**

2. **Subir archivos:**
   - Haz clic en **"Add file"** y selecciona **"Upload files"**.
   - Arrastra y suelta los archivos o selecciona desde tu computadora (por ejemplo, `prueba.c`).
<img width="1173" height="326" alt="subirArchivos" src="https://github.com/user-attachments/assets/b22dbcf6-63c2-4c4e-bc8a-0767f0e6f1e9" />

3. **Confirmar los cambios:**
   - Escribe un mensaje en el campo "Commit changes", como "Subiendo archivo de prueba".
   - Asegúrate de seleccionar **"Commit directly to the main branch"**.
   - Haz clic en **"Commit changes"**.
<img width="1182" height="404" alt="commitChanges" src="https://github.com/user-attachments/assets/60b90e04-e16d-473f-95cc-4e20376bb04a" />

### Paso 3: Ver Archivos y Commits

1. **Ver archivos:**  
   En el repositorio, aparecerán los archivos que has subido. Puedes hacer clic sobre ellos para ver su contenido.

2. **Ver historial de commits:**  
   Haz clic en la pestaña **"Commits"** para ver una lista con todos los cambios realizados, quién los hizo y cuándo.
<img width="860" height="318" alt="historialCommits" src="https://github.com/user-attachments/assets/bfa6b8f8-e5c9-438d-bb46-746d4c7830cf" />

### Paso 4: Crear y Administrar Ramas (Branches)

Las ramas permiten trabajar en nuevas características o correcciones sin afectar la versión principal.

1. **Crear una nueva rama:**
   - En el repositorio, haz clic en el menú desplegable que dice "main".
   - Escribe un nombre para la nueva rama y haz clic en **"Create branch"**.
<img width="867" height="399" alt="nuevaRama" src="https://github.com/user-attachments/assets/de8f7810-59c3-4c20-8f8d-6cde9d4d7c03" />

2. **Trabajar en la nueva rama:**  
   Desde esta rama puedes agregar o modificar archivos sin afectar la rama principal.

<img width="869" height="447" alt="cambiosRama" src="https://github.com/user-attachments/assets/963b55dc-0ac8-45c0-a90c-28369e6edc87" />
<img width="860" height="534" alt="cambiosRamaDesc" src="https://github.com/user-attachments/assets/4ee73475-f7ab-4c17-a8f2-7fa856d27dfc" />
3. **Fusionar ramas (merge):**
   - Ve a la pestaña **"Pull requests"**.
   - Haz clic en **"New pull request"**.
   - Elige las ramas que quieres fusionar.
   - Crea el pull request y, tras revisar los cambios, haz clic en **"Merge pull request"**.

### Paso 5: Configuración y Permisos

1. **Configuración del repositorio:**
   - Haz clic en la pestaña **"Settings"**.
   - Aquí puedes cambiar el nombre, la visibilidad o eliminar el repositorio.

2. **Agregar colaboradores:**
   - En la sección **"Collaborators"**, puedes invitar a otros usuarios para que colaboren en tu proyecto.
   - Define sus permisos (lectura, escritura, administración).

---

## Conclusiones

GitHub es una herramienta esencial para todo desarrollador, ya que permite trabajar de manera colaborativa, llevar un control detallado de versiones y mantener organizados los proyectos de software. Con esta guía básica, ya puedes empezar a crear tus propios repositorios, subir archivos, gestionar ramas y colaborar con otros usuarios de forma efectiva.
